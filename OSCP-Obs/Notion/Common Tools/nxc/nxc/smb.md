## `--lsa` 是什么？

`--lsa` 用于从目标主机的 **LSA（Local Security Authority）Secrets** 中提取信息。

高层概念说明（不涉及可滥用的操作细节）：

- LSA Secrets 存储系统级的敏感配置，例如服务账户的凭证片段、系统密钥等。
    
- 在授权测试中，分析这些数据可以帮助识别 **错误配置** 或 **过度权限** 情况。
    
- 此选项通常需要具有较高权限才能读取相关信息。

## `--ntds` 是什么？ NT Directory Services

`--ntds` 用于在 **域控制器（Domain Controller）** 上提取与 `NTDS.dit`（Active Directory 数据库）相关的信息。

概念上要点如下：

- `NTDS.dit` 是 Active Directory 的核心数据库，保存用户账户、组、哈希等目录对象。
    
- 在合法的安全评估中访问它可以帮助验证 **AD 安全性**、**密码策略强度**、**访问控制配置** 等。
    
- 由于其敏感性，此功能通常需要域管理员级别权限，并且必须在严格授权范围内执行。
