- **SeMachineAccountPrivilege（或“Add workstations to domain”）**  
    这是 Windows/AD 中的一个权限/特权，允许帐号创建计算机（machine）账户并将工作站加入域。默认情况下，域中的普通认证用户通常被允许创建**最多 10** 个计算机帐户（这一限制可由域策略调整）。它是一个**权限/能力**，不是漏洞。
    
- **CVE-2021-42278 与 CVE-2021-42287**  
    这是微软在 2021 年披露的两个 Active Directory 漏洞（补丁编号分别在当时的安全公告中提及）。它们并不是权限，而是**实现缺陷/安全漏洞**，攻击者可以利用这些漏洞在特定条件下提升控制权限或欺骗/滥用计算机账户认证逻辑，从而进行横向移动或权限提升。

## 2. 二者如何关联（为什么看起来“像一回事”）

这两枚 CVE 的连锁利用链依赖于**能够创建或控制计算机账户**这个前提——而 SeMachineAccountPrivilege（或域中允许普通用户创建少量机器账户的默认策略）恰恰可能给攻击者提供了那种能力。

典型思路（高层次）：

1. 攻击者利用已有的创建计算机账户的能力（来源可能是 SeMachineAccountPrivilege 或域默认策略）创建或注册一个特定名称/格式的计算机账户。
    
2. 利用 CVE 漏洞绕过 AD 的某些校验或认证限制，使新创建的计算机账户能够以不正当方式代表/冒充其他计算机，从而获取更高权限或可横向移动的凭据。
    
3. 最终实现对目标的权限提升或域内扩散。
    

所以：**SeMachineAccountPrivilege 是一个便利条件（权限），CVE-2021-42278/42287 是可被利用的漏洞**。两者结合会构成危险的攻击路径；但没有漏洞，仅有权限不一定能达成同样效果；没有权限但有别的利用向量也可能利用漏洞。
